# Phase 01.1 Plan 01: Fix Direct ATEC Site-Root Upload Path Summary

**One-liner:** Auto-create `Unit [N]` subfolder under site root when Telegram user presses `u` without navigating deeper, preventing photos from landing in the site root.

## What Was Built

Fixed `_handle_nav_reply` in `utils/telegram_bot.py` to handle the case where a user confirms the upload destination at the site root (empty `segments`). Previously, photos would be saved directly under `Sites/[site_name]/` with no unit folder structure and no template was saved. Now:

1. `_find_or_create_folder` is called to create (or find existing) `Unit [unit_number]` under the site root
2. `SITES.learn()` is called with the single-segment template `["Unit [unit_number]"]` so future bookouts resolve the unit folder without interactive navigation
3. The confirmation message shows `Unit N (auto-created)` instead of `(site root)`
4. Error handling cleans up temp files and clears state if the Drive call fails
