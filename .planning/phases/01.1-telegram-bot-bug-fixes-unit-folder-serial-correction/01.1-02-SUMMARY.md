# Phase 01.1 Plan 02: Serial Number Correction Step Summary

**One-liner:** New `STEP_SERIAL_CORRECTION` FSM step intercepts not-found serials before swap mode, letting technicians correct OCR misreads via a chat reply with automatic stock re-lookup.

## What Was Built

Added a serial correction gate between stock lookup failure and swap confirmation in the Telegram bot bookout flow.

### Task 1: STEP_SERIAL_CORRECTION constant (`utils/telegram_state.py`)

Added `STEP_SERIAL_CORRECTION = "serial_correction"` after `STEP_SWAP_CONFIRM` in the step constants block. No other changes to the file.

### Task 2: Serial correction wired into `utils/telegram_bot.py`

Five changes made to the bot:

1. **Import** — `STEP_SERIAL_CORRECTION` added to the `from utils.telegram_state import (...)` block.

2. **`format_serial_correction_prompt(serial, item_code)`** — new formatter that shows the not-found serial as Markdown inline code and prompts for a correction or "swap".

3. **`_process_bookout` swap branch** — replaced the direct `STEP_SWAP_CONFIRM` transition with `STEP_SERIAL_CORRECTION`. The index of the first not-found item is stored in `state["_correction_serial_index"]` so the handler knows which item is under correction.

4. **`on_message` dispatcher** — new branch for `STEP_SERIAL_CORRECTION` (before `STEP_SWAP_CONFIRM`) dispatches to `_handle_serial_correction`; `/cancel` clears state as usual.

5. **`_handle_serial_correction` + `_advance_serial_correction`** — two new async functions:
   - "swap" reply marks the current item as swap and calls `_advance_serial_correction`.
   - Any other text is treated as a corrected serial; `find_serial_number` is called via `asyncio.to_thread`. If found, the item serial is updated and `is_swap` cleared. If still not found, falls back to `STEP_SWAP_CONFIRM`.
   - `_advance_serial_correction` scans for additional not-found items beyond the current index and prompts for the next, or calls `_continue_after_swap_confirm` when all are resolved.
